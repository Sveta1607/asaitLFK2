# Пошаговая инструкция: переключение на PostgreSQL

Для новичков. Выполняйте шаги по порядку.

---

## Шаг 1. Установить PostgreSQL на компьютер

1. Скачайте установщик с официального сайта:  
   **https://www.postgresql.org/download/windows/**  
   (выберите последнюю стабильную версию, например 16 или 17).

2. Запустите установщик. Во время установки:
   - Запомните или запишите **пароль суперпользователя (postgres)** — он понадобится позже.
   - Порт оставьте по умолчанию: **5432**.
   - Остальное можно оставить как есть.

3. Дождитесь окончания установки. PostgreSQL будет работать как служба в фоне.

---

## Шаг 2. Создать базу данных для проекта

1. Откройте программу **pgAdmin** (она ставится вместе с PostgreSQL)  
   или **Командную строку** (cmd) / **PowerShell**.

2. **Вариант А — через pgAdmin (проще для новичка):**
   - Запустите pgAdmin.
   - Слева раскройте: Servers → PostgreSQL → введите пароль пользователя postgres.
   - Правой кнопкой по **Databases** → **Create** → **Database**.
   - В поле **Database** введите имя: **lfk_app**.
   - Нажмите **Save**. База создана.

3. **Вариант Б — через командную строку:**
   - Откройте cmd или PowerShell.
   - Выполните (подставьте свой пароль вместо `ВАШ_ПАРОЛЬ`):
   ```bash
   "C:\Program Files\PostgreSQL\16\bin\psql.exe" -U postgres -c "CREATE DATABASE lfk_app;"
   ```
   - Если PostgreSQL стоит в другой папке (например, 17), замените `16` на нужную версию в пути.

---

## Шаг 3. Узнать данные для подключения

Для строки подключения нужны:

| Что           | Обычно так и есть        |
|---------------|---------------------------|
| **Хост**      | `localhost`               |
| **Порт**      | `5432`                    |
| **Имя БД**    | `lfk_app` (как создали)   |
| **Пользователь** | `postgres`             |
| **Пароль**    | тот, что задали при установке |

Строка подключения будет такой (подставьте свой пароль):

```
postgresql+psycopg2://postgres:ВАШ_ПАРОЛЬ@localhost:5432/lfk_app
```

Пример: если пароль `mypass123`:

```
postgresql+psycopg2://postgres:mypass123@localhost:5432/lfk_app
```

---

## Шаг 4. Прописать подключение в проекте

1. Откройте папку **backend** вашего проекта.

2. Откройте файл **`.env`** в блокноте или в Cursor.

3. Найдите закомментированную строку с `DATABASE_URL` (она начинается с `#`).

4. Добавьте **новую строку** (без решётки в начале) и вставьте свою строку подключения с вашим паролем:

   ```env
   DATABASE_URL=postgresql+psycopg2://postgres:ВАШ_ПАРОЛЬ@localhost:5432/lfk_app
   ```

   Замените `ВАШ_ПАРОЛЬ` на реальный пароль пользователя postgres.

5. Сохраните файл `.env`.

**Важно:** если в пароле есть спецсимволы (например `#`, `@`, `%`), закодируйте их в URL или придумайте пароль без таких символов.

---

## Шаг 5. Убедиться, что установлен драйвер для PostgreSQL

В проекте уже есть зависимость в **backend/requirements.txt**:

```
psycopg2-binary>=2.9.0
```

Если вы раньше ставили зависимости, выполните в папке **backend** в терминале:

```bash
python -m pip install -r requirements.txt
```

Так вы гарантированно установите/обновите драйвер для PostgreSQL.

---

## Шаг 6. Запустить бэкенд заново

1. Если бэкенд (uvicorn) уже запущен — остановите его (в терминале: **Ctrl+C**).

2. Запустите снова из папки **backend**:

   ```bash
   cd backend
   python -m uvicorn main:app --host 127.0.0.1 --port 3000
   ```

3. При первом запуске с PostgreSQL:
   - приложение само создаст таблицы (users, news, slots, bookings);
   - выполнится seed: добавятся тестовые пользователи, новости, слоты и одна запись.

Дальше все данные будут храниться уже в PostgreSQL.

---

## Проверка, что всё работает

- Откройте сайт (фронтенд) и убедитесь, что новости отображаются.
- Войдите как специалист, добавьте новость, обновите страницу — новость должна остаться (она уже в PostgreSQL).
- В pgAdmin можно открыть базу **lfk_app** → **Schemas** → **public** → **Tables** и посмотреть таблицы и данные.

---

## Если что-то пошло не так

- **Ошибка «connection refused»** — PostgreSQL не запущен. Запустите службу PostgreSQL в Windows (Службы) или перезапустите компьютер после установки.
- **Ошибка «password authentication failed»** — неверный пароль в `DATABASE_URL` в `.env`.
- **Ошибка «database lfk_app does not exist»** — база не создана. Вернитесь к шагу 2 и создайте базу **lfk_app**.

Если опишете текст ошибки из терминала или браузера, по нему можно подсказать точнее.
